# Akiya Portal Agent Rules

このリポジトリでAIが手動で記事を書く、SEO/AIO/GEO施策を行う、既存記事を改善する場合は、以下を必ず守る。自動生成ジョブは止めないが、生成プロンプト・記事構成を改善するときはこの基準に合わせる。

## 前提

- ゴールはGoogle AI Overview、ChatGPT Search、Perplexity、GeminiなどのAI検索に引用されること。
- 最適化単位はページ全体ではなく、AIが抜き出せるパッセージである。
- 完全一致キーワードの反復ではなく、読者の自然な質問に対する明確な回答を作る。

## 対象サイト

- サイトURL: 未確認
- サービス概要: 英語圏ユーザー向けに、日本の空き家購入、空き家バンク、リノベーション、地方移住、外国人購入プロセスを案内するサイト。
- 想定読者: 日本の空き家購入を検討する外国人、海外在住者、デジタルノマド、リタイア層、地方移住希望者。

## 禁止事項

- 「地域名＋業種＋キーワード」の機械的な羅列を見出しや本文に詰めない。
- 完全一致キーワードの繰り返し、キーワードスタッフィングをしない。
- 「Akiyaとは？」だけの薄い定義記事を作らない。
- 検索ボリューム起点だけで構成を組まない。
- Akiya Portal固有の購入手順、費用、リスク、地域差、外国人向け注意点がない一般論だけの記事を出さない。

## 手動記事・改善時の工程

1. 対象URLと主要下層ページを読み、読者の悩み、提供価値、固有情報、不足情報を整理する。
2. 読者がAIに聞く自然な質問を15〜20個作り、情報収集・比較検討・今すぐ行動に分類する。
3. 質問を3〜6本の記事案へクラスタリングし、タイトル、カバー質問、記事タイプを提示する。
4. 新規記事の執筆に進む前に、ユーザー承認を待つ。
5. 承認後の記事はQuick Answer、質問型見出し、比較表、番号付き手順、FAQ、JSON-LDを入れる。

## 提出前チェック

- キーワード羅列になっていない。
- 各セクション冒頭200字以内で結論がわかる。
- 見出しが自然な質問文になっている。
- Akiya固有の具体情報、費用、地域、リスク、外国人向け注意点が入っている。
- FAQとJSON-LDがある。
